# XingHui WebSocket 服务器 (Cloud Store & RtCVS)
一个基于 WebSocket 的实时云端数据同步服务器，主要面向图形化玩家提供**持久化的键值云存储**和**实时同步的云变量**两项核心服务。

## ✨ 核心功能
1.  **持久化云存储**
    *   支持用户通过 `dump`/`write`/`w` 指令上传数据。
    *   支持通过 `load`/`read`/`r` 指令读取已存储的数据。
    *   支持通过 `remove`/`delete`/`del` 指令删除指定数据。
    *   数据以文本文件形式持久化保存在服务器，按用户 Token 命名空间隔离。
2.  **实时云变量**
    *   支持通过 `CV_dump`/`CV_write`/`CV_w` 指令设置或更新变量。
    *   一旦变量更新，服务器会自动将完整的变量集广播给所有声明了 `"type": "RtCVS"` 的客户端，实现多客户端间的**实时状态同步**。
    *   单用户下所有云变量总大小限制为 32KB。
3.  **完善的用户与配额管理 (UAT 系统)**
    *   每个用户拥有独立的 Token 和配置，包括每日调用次数配额、存储空间限额等。
    *   服务器每日自动重置所有用户的可用调用次数 (`total_uses`)。
    *   自动计算并跟踪每个用户的已用存储空间 (`used_storage`)。
    *   提供 `i_me` 指令供客户端查询自己的账户状态。
4.  **服务器状态监控**
    *   提供 `i_sever` 指令，返回服务器的 CPU、内存、磁盘占用等性能信息。
5.  **详尽的日志与容错**
    *   所有连接、指令、错误均被记录到按日期分隔的日志文件中 (`_all.log`, `_info.log`, `_warning.log`)。
    *   对非法请求、配额不足、存储超限等情况，会返回友好（且略带俏皮）的错误提示。
      
## 🛠 技术架构
*   **通信协议**：WebSocket (基于 `SimpleWebSocketServer` 库)
*   **数据格式**：JSON
*   **数据持久化**：文件系统 (按用户 Token 分目录存储 `.txt` 文件)
*   **并发模型**：多线程 (用于后台配额检查任务)
*   **核心逻辑**：连接管理、消息路由、实时广播、配额校验、日志记录

## 📁 项目结构 (摘要)
*   `XingHui_WS/`
    *   `log/` - 存放按日期生成的运行日志。
    *   `data/` - 存放用户的持久化数据文件，按 Token 分目录。
    *   `config/`
        *   `UAT.json` - 存储所有用户的账户令牌、配额及使用信息。
        *   `timelast_updated.txt` - 记录上次配额重置日期。

## 🚀 快速开始
1.  **依赖安装**：`pip install SimpleWebSocketServer psutil`
2.  **修改**：修改代码文件中的`父目录`变量为自己的实际目标目录
3.  **运行服务器**：`python XingHui_WS云存储服务器.py`
4.  **客户端连接**：通过 WebSocket 连接到服务器地址 (默认: `ws://127.0.0.1:55550`)。您可以使用内网穿透(FRP)将内网映射至公网(内网参数:ip'127.0.0.1',端口'55550',协议'TCP')
5.  **首次交互**：连接后，服务器会发送欢迎语和免责声明。客户端发送的任何有效指令必须为 JSON 格式，且包含有效的 `"UAT"` 字段。

**示例客户端请求 (存储数据)：**
```json
{
  "dump":["存储文件名","数据"]
  "UAT":"token"
}
```
